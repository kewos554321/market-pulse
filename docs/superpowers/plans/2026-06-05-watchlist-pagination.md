# Watchlist Pagination Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install Tailwind CSS + shadcn/ui into `web/`, then add client-side pagination (prev/next + customizable page size) to the Watchlist page.

**Architecture:** Two phases — (1) tooling setup: Tailwind v4 via `@tailwindcss/vite` plugin + shadcn/ui initialized with path aliases; (2) feature: add `currentPage`/`pageSize` state to `Watchlist.tsx`, slice `filteredItems` for display, and render a pagination bar using shadcn/ui `Pagination` + `Select`. No API changes.

**Tech Stack:** Tailwind CSS v4, shadcn/ui (Radix UI-based), Vite 5, React 18, TypeScript 5

- [ ] **Step 7: Add the pagination bar below the stock list**

Find the closing `</div>` of the `{/* Stock list */}` section. It is the last `</div>` in the return, just before the outer `</div>`. Add the pagination bar **after** the stock list `</div>`:

      {/* Pagination bar */}

                  className={currentPage === totalPages ? 'pointer-events-none opacity-40' : 'cursor-pointer'}
                />
              </PaginationItem>
            </PaginationContent>
          </Pagination>
